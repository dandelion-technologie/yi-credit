# v0.1 静态展示网页优化清单与方案

日期：2026-07-17

## 版本目标

建立静态展示网页的首版优化基线，明确后续页面搭建、视觉设计、内容组织、性能、SEO、响应式和上线验收的检查项。

## 当前状态

- 项目为 `yi-credit` monorepo。
- Web 端位于 `apps/web`，使用 Next.js App Router。
- 已有 Tailwind、next-intl、基础布局和中英文消息文件。
- 本版本已建立文档化优化框架，并落地基础品牌色配置。
- 已补齐首页 locale 静态化设置，`/en` 与 `/zh` 可完成静态生成。

## 优化清单

- [ ] 明确首屏核心信息：品牌名、业务定位、核心卖点、主行动入口。
- [ ] 梳理静态展示网页的主要版块顺序。

- [ ] 减少营销空话，优先展示可信信息、服务范围、流程和联系方式。

- [ ] 页面风格保持清晰、专业、可读，避免单一色系堆叠。

- [ ] 导航、语言切换、联系入口等基础交互清晰可用。

## 优化方案

### 1. 页面结构方案

首版建议采用以下结构：

1. 首屏：品牌名、定位说明、核心卖点、主要行动按钮。
2. 信任信息：资质、数据、客户类型或合作亮点。
3. 服务介绍：按业务场景拆分 3 到 5 个服务模块。
4. 流程说明：用清晰步骤表达从咨询到交付的路径。
5. 常见问题：减少用户决策疑虑。
6. 联系区域：电话、邮箱、表单入口或外部联系方式。

### 2. 技术实现方案

- 优先在 `apps/web/app/[locale]/page.tsx` 实现静态展示首页。
- 文案放入 `apps/web/messages/zh.json` 和 `apps/web/messages/en.json`。
- 视觉样式沿用 Tailwind，避免新增复杂 UI 依赖。
- 页面颜色统一使用 `tailwind.config.ts` 中的 `yi-*` 品牌色。
- 品牌调色板记录在 [color-palette-v0.1.md](./color-palette-v0.1.md)。
- 页面组件可先在单页内组织，待内容稳定后再拆成局部组件。
- 图片资产统一放入 `apps/web/public` 或项目现有静态资源目录。

### 3. 质量控制方案

开发完成后至少执行：

```bash
npm run lint
npm run typecheck
npm run build
```

必要时补充浏览器人工检查：

- 桌面宽度：1440px
- 平板宽度：768px
- 移动宽度：390px

## 验收标准

- 首页能清晰表达品牌、业务和联系方式。
- 中英文内容结构一致，无明显缺漏。
- 页面在移动端和桌面端无文字溢出、错位或遮挡。
- 构建、Lint、类型检查通过。
- `docs/` 中存在当前版本的优化清单和方案。
